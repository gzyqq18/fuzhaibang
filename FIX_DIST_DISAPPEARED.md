# dist 文件夹文件被清空的问题

## ❌ 问题

dist 文件夹里的文件莫名其妙消失了，比如：
- 抖音开发者工具导入后，文件就没了
- 执行了某个命令后，文件被清空了
- 重启电脑后，文件不见了

---

## 🔍 原因分析

### 原因 1：抖音开发者工具的"预编译"设置

**抖音开发者工具有一个设置**：`启用预编译`

**如果这个设置被启用**：
- 抖音开发者工具会"预编译"项目
- 这个过程中可能会清空 dist 目录
- 然后重新生成编译后的文件

**影响**：
- 如果编译失败，dist 目录就会变成空的
- 如果设置不正确，文件会反复消失

### 原因 2：清理命令被执行

可能执行了以下命令之一：

```bash
# 清理编译输出
pnpm clean

# 清理 dist 目录
rm -rf dist

# 清理所有构建文件
pnpm run clean
```

### 原因 3：文件夹权限问题

如果 dist 目录的权限不正确，可能导致：
- 文件无法正确写入
- 文件被系统自动清理
- 抖音开发者工具无法访问

### 原因 4：同步软件或云盘

如果你的项目在以下位置：
- OneDrive 同步的文件夹
- Google Drive 同步的文件夹
- 其他云同步盘

**问题**：
- 云盘可能会认为编译后的文件是"临时文件"
- 自动清理这些文件
- 导致 dist 目录变空

---

## ⚡ 快速解决方案

### 步骤 1：重新编译项目

```powershell
cd Desktop\projects
pnpm build:ttapp
```

### 步骤 2：验证编译结果

```powershell
# Windows
dir dist

# Mac/Linux
ls -la dist
```

**应该看到**：
```
app.js
app.json
app.wxss
pages/
...
```

### 步骤 3：保护 dist 目录（重要！）

#### 方法 1：使用 .gitignore 避免被忽略

**问题**：如果项目使用了 Git，dist 目录可能被忽略了

**检查**：
```powershell
# 查看 .gitignore 文件
type .gitignore

# 如果看到以下内容，说明 dist 被忽略了：
# dist/
```

**解决方案**：
- 这不是问题，dist 目录本来就不应该提交到 Git
- 每次拉取代码后，都需要重新编译

#### 方法 2：修改抖音开发者工具设置

**关闭"启用预编译"**：

1. 打开抖音开发者工具
2. 点击右上角"详情"
3. 找到"本地设置"
4. ❌ 取消勾选"启用预编译"

#### 方法 3：避免在云同步盘中工作

**检查项目位置**：
```powershell
# 查看当前目录
pwd
```

**如果路径包含以下内容，建议移动项目**：
- `OneDrive`
- `Google Drive`
- `Dropbox`
- `百度网盘`

**移动项目到本地磁盘**：
```powershell
# 1. 复制项目到桌面
cp -r /path/to/onedrive/projects ~/Desktop/

# 2. 进入新目录
cd ~/Desktop/projects

# 3. 重新编译
pnpm build:ttapp
```

#### 方法 4：设置 dist 目录为只读（临时保护）

**Windows**：
```powershell
# 右键点击 dist 文件夹 > 属性 > 只读
# 或者使用命令
attrib +R dist
```

**注意**：这会阻止写入，每次编译前需要取消只读属性。

## 🚨 重要提示

### ⚠️ dist 目录是编译输出，不应该手动管理

**正确的理解**：
- dist 目录是**编译生成的临时文件**
- 不应该手动修改或管理
- 每次编译都会重新生成
- 可以随时删除和重新生成

**正确的做法**：
1. 不要手动修改 dist 目录中的文件
2. 不要将 dist 目录提交到 Git
3. 每次修改代码后，重新编译
4. 如果文件消失，重新编译即可

### ⚠️ 抖音开发者工具会自动刷新

**正常流程**：
1. 编译项目：`pnpm build:ttapp`
2. 导入到抖音开发者工具
3. 修改代码
4. 重新编译：`pnpm build:ttapp`
5. 抖音开发者工具会自动检测到变化并刷新

**不需要**：
- 手动刷新抖音开发者工具
- 手动复制文件到 dist 目录
- 手动删除 dist 目录

## 📋 预防措施

### 1. 关闭抖音开发者工具的预编译

```
抖音开发者工具 > 详情 > 本地设置 > ❌ 取消勾选"启用预编译"
```

### 2. 使用开发模式

```bash
coze dev
```

优势：
- 自动编译
- 自动刷新
- 不会出现文件消失的问题

### 3. 避免在云同步盘中工作

确保项目在本地磁盘上：
- ✅ C:\Users\YourName\Desktop\projects
- ✅ /Users/YourName/Desktop/projects
- ❌ C:\Users\YourName\OneDrive\projects
- ❌ /Users/YourName/OneDrive/projects

### 4. 不要手动管理 dist 目录

**不要做**：
- ❌ 手动删除 dist 目录
- ❌ 手动添加文件到 dist 目录
- ❌ 手动修改 dist 目录中的文件
- ❌ 将 dist 目录提交到 Git

**应该做**：
- ✅ 使用 `pnpm build:ttapp` 编译
- ✅ 修改代码后重新编译
- ✅ 依赖编译命令生成文件

## 🐛 如果问题持续出现

### 检查清单

- [ ] 项目不在云同步盘中
- [ ] 抖音开发者工具关闭了"启用预编译"
- [ ] 没有手动删除 dist 目录

3. **文件夹权限问题**

2. **使用开发模式（coze dev）**
3. **避免在云同步盘中工作**
4. **不要手动管理 dist 目录**

1. **重新编译**：`pnpm build:ttapp`
2. **验证结果**：`dir dist`
3. **抖音开发者工具会自动刷新**
