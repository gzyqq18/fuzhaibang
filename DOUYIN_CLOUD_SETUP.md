# 抖音云后端服务配置指南

## 问题描述

当前小程序无法访问后端服务，报错：
- "分类列表: undefined"
- "智能问答结果: undefined"
- "TypeError: Cannot read properties of undefined (reading 'answer')"

## 根本原因

`.env.local` 文件中配置的 `PROJECT_DOMAIN` 是 Coze 开发环境的域名，不是抖音云后端服务的域名。

## 解决方案

### 步骤 1：获取抖音云应用的公网域名

1. **登录抖音云控制台**
   - 访问：https://console.volces.com/douyincloud

2. **找到应用「fuzhaibang」**

3. **查看应用详情**
   - 在应用详情页，找到「访问地址」或「公网域名」
   - 域名格式通常为：`https://app-xxx.douyincloud.com`

4. **记录域名**
   - 例如：`https://app-1234567890.douyincloud.com`

### 步骤 2：更新 `.env.local` 文件

```bash
# 修改前
PROJECT_DOMAIN=https://64dacf48-a65a-4f6c-9fb4-f05851730eb1.dev.coze.site

# 修改后（替换为实际的抖音云域名）
PROJECT_DOMAIN=https://app-1234567890.douyincloud.com
```

### 步骤 3：在小程序后台配置域名白名单

1. **登录抖音开放平台**
   - 访问：https://developer.open-douyin.com

2. **进入小程序管理后台**
   - 找到小程序「负债人自救指南」（appid: tt27f0514db990b96201）

3. **配置服务器域名**
   - 导航：开发 → 开发管理 → 开发设置 → 服务器域名
   - 在 request 域名中添加抖音云的公网域名（不带 https://）
   - 例如：`app-1234567890.douyincloud.com`

4. **保存配置**

### 步骤 4：重新编译和上传小程序

```bash
# 编译小程序
pnpm build:weapp

# 或使用开发者工具重新上传代码
```

## 临时测试方案（不推荐）

如果暂时无法获取抖音云域名，可以使用以下临时方案：

### 方案 A：禁用域名检查（仅开发环境）

修改 `project.config.json`：

```json
{
  "setting": {
    "urlCheck": false  // 禁用域名检查
  }
}
```

⚠️ **警告**：此方案仅适用于开发环境，正式发布前必须配置正确的域名白名单。

### 方案 B：使用本地开发环境（仅测试）

1. 在本地启动后端服务：
```bash
pnpm dev:server
```

2. 修改 `.env.local`：
```
PROJECT_DOMAIN=http://localhost:3000
```

3. 使用开发者工具的「真机调试」功能测试

⚠️ **警告**：此方案仅适用于真机调试，发布版本无法访问 localhost。

## 验证配置

配置完成后，可以通过以下方式验证：

1. **查看网络请求日志**
   - 在开发者工具中，查看 Console 面板的网络请求日志
   - 查看请求 URL 是否正确，响应状态码是否为 200

2. **测试接口**
   - 访问：`https://app-xxx.douyincloud.com/api/health`
   - 应该返回：`{"status":"ok"}`

3. **测试小程序功能**
   - 打开小程序
   - 首页应该显示分类卡片
   - 智能问答功能应该正常工作

## 常见问题

### Q1: 如何找到抖音云应用的公网域名？

**A:** 在抖音云控制台的应用详情页，查找以下位置：
- 「访问地址」区域
- 「应用信息」标签页
- 「部署」标签页的「访问方式」

### Q2: 域名白名单配置后多久生效？

**A:** 通常在 5-10 分钟内生效，最长不超过 30 分钟。

### Q3: 报错 "不在以下 request 合法域名列表中" 怎么办？

**A:** 说明域名没有在小程序后台的白名单中，请检查：
1. 域名是否正确（不要带 https://）
2. 是否已经保存配置
3. 配置是否已生效（等待 5-10 分钟）

### Q4: 网络请求返回 undefined 怎么办？

**A:** 检查以下几点：
1. PROJECT_DOMAIN 是否配置正确
2. 域名是否在小程序白名单中
3. 后端服务是否正常运行
4. 网络连接是否正常
