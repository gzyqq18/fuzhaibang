# 抖音小程序网络配置指南

## 问题描述
导入抖音开发者工具后，精选资料分类显示空白，无法加载分类数据。

## 原因分析
项目通过 API 接口 `/api/knowledge/categories` 获取分类数据，但在抖音开发者工具中：
1. `PROJECT_DOMAIN` 环境变量可能未配置
2. 网络请求被域名校验拦截
3. 后端服务未启动或无法访问

## 解决方案

### 方案一：关闭域名校验（推荐，最简单）

**步骤：**

1. 打开抖音开发者工具
2. 点击右上角「详情」按钮
3. 切换到「本地设置」选项卡
4. 勾选「不校验合法域名、web-view（业务域名）、TLS 版本以及 HTTPS 证书」
5. 重新编译项目

**优点：**
- 配置简单，一键解决
- 适合开发和测试阶段

**注意：**
- 正式发布时需要配置合法域名
- 此选项仅对本地开发有效

### 方案二：配置环境变量

**步骤：**

1. 在项目根目录创建 `.env.local` 文件
2. 添加以下内容：
   ```env
   PROJECT_DOMAIN=http://localhost:3000
   ```
3. 重新编译项目：
   ```bash
   pnpm build
   ```

**优点：**
- 配置了明确的 API 地址
- 适合有后端服务的开发环境

**注意：**
- 需要确保后端服务已启动
- 在抖音开发者工具中仍需关闭域名校验

### 方案三：配置服务器域名（正式发布用）

**步骤：**

1. 登录 [抖音开放平台](https://developer.open-douyin.com/)
2. 进入应用管理
3. 在「开发」->「服务器域名」中配置：
   - request 合法域名：`https://your-domain.com`
   - uploadFile 合法域名：`https://your-domain.com`
   - downloadFile 合法域名：`https://your-domain.com`

**优点：**
- 正式发布必需配置
- 符合抖音平台规范

**注意：**
- 域名必须备案
- 必须使用 HTTPS

### 方案四：本地启动后端服务

如果需要完整的后端功能（智能问答、内容解锁等），需要启动后端服务：

**步骤：**

1. 确保数据库已配置（Supabase）
2. 启动后端服务：
   ```bash
   cd server
   pnpm install
   pnpm dev
   ```
3. 后端服务默认运行在 `http://localhost:3000`

**验证后端服务：**
```bash
curl http://localhost:3000/api/knowledge/categories
```

## 推荐配置（开发阶段）

### 最小配置（无需后端）
1. 关闭域名校验（方案一）
2. 使用项目内置的分类数据

### 完整配置（需要后端）
1. 创建 `.env.local` 文件（方案二）
2. 关闭域名校验（方案一）
3. 启动后端服务（方案四）

## 验证方法

**检查分类是否加载：**
1. 打开抖音开发者工具
2. 进入首页
3. 查看是否显示 4 个分类卡片：
   - 💰 债务管理
   - 🧠 心理疏导
   - ⚖️ 法律知识
   - 📋 资源导航

**查看控制台日志：**
1. 打开开发者工具控制台
2. 查看是否有以下输出：
   - ✅ `分类列表: [...]` - 加载成功
   - ❌ `加载分类失败: ...` - 加载失败

## 常见问题

### Q1: 关闭域名校验后还是无法加载分类？
**A**: 检查后端服务是否启动，或者项目是否使用了 fallback 数据。

### Q2: 正式发布时域名校验怎么办？
**A**: 必须在抖音开放平台配置合法的服务器域名。

### Q3: 如何查看分类数据是否从 API 加载？
**A**: 打开控制台，查看是否有「分类列表」的日志输出。

### Q4: 不想启动后端服务，能正常使用吗？
**A**: 可以，项目已经配置了 fallback 数据，即使没有后端也能显示分类。

## 技术细节

**网络请求流程：**
```
前端代码
  ↓ Network.request('/api/knowledge/categories')
  ↓ 添加 PROJECT_DOMAIN 前缀
  ↓ 实际请求：PROJECT_DOMAIN/api/knowledge/categories
  ↓
后端服务（如果配置了 PROJECT_DOMAIN）
或
本地 fallback 数据（如果没有配置 PROJECT_DOMAIN）
```

**环境变量优先级：**
```
process.env.PROJECT_DOMAIN
↓ (如果为空)
process.env.COZE_PROJECT_DOMAIN_DEFAULT
↓ (如果为空)
'' (空字符串)
```
